Локальный стек

Базы данных и админки

Быстрые ссылки на админки и примеры строк подключения: с хоста (имя db.local) и из контейнеров в сети databases_net. На вкладках — готовые фрагменты для Cursor MCP (~/.cursor/mcp.json); полный пример и MCP_DOCKER — в README репозитория. Как включить домен и портал — там же.

PostgreSQL

Реляционная БД. Схемы и SQL — в pgAdmin.

Открыть pgAdmin Документация
С хоста (db.local)
postgresql://postgres:postgres@db.local:5432/postgres
Контейнер в сети databases_net
URI
postgresql://postgres:postgres@postgres:5432/postgres
JDBC
jdbc:postgresql://postgres:5432/postgres?user=postgres&password=postgres
Cursor MCP (~/.cursor/mcp.json)

Вставьте внутрь mcpServers. Процесс MCP в Docker — в DATABASE_URI указывается host.docker.internal, чтобы из контейнера MCP достучаться до порта на хосте.

"postgres-local": {
  "command": "docker",
  "args": [
    "run", "-i", "--rm",
    "--add-host=host.docker.internal:host-gateway",
    "-e", "DATABASE_URI",
    "crystaldba/postgres-mcp:latest",
    "--access-mode=restricted"
  ],
  "env": {
    "DATABASE_URI": "postgresql://postgres:postgres@host.docker.internal:5432/postgres"
  }
},

MongoDB

Документы и коллекции — в mongo-express.

Открыть mongo-express Документация
С хоста (db.local)
mongodb://admin:admin@db.local:27017/?authSource=admin
Контейнер в сети databases_net
mongodb://admin:admin@mongodb:27017/?authSource=admin
Cursor MCP (~/.cursor/mcp.json)

Локально через npx; строка подключения как у MONGODB_URI (можно 127.0.0.1 или db.local).

"mongodb": {
  "command": "npx",
  "args": ["-y", "mongodb-mcp-server"],
  "env": {
    "MDB_MCP_CONNECTION_STRING": "mongodb://admin:admin@127.0.0.1:27017/?authSource=admin",
    "MDB_MCP_READ_ONLY": "true"
  }
},

Elasticsearch

Поиск и индексы. Обзор — в Kibana.

Открыть Kibana Документация
С хоста (db.local)
http://elastic:elastic_local@db.local:9200
Контейнер в сети databases_net
http://elastic:elastic_local@elasticsearch:9200
Cursor MCP (~/.cursor/mcp.json)

MCP в Docker; в ES_URL — host.docker.internal (не 127.0.0.1: внутри контейнера MCP это другой localhost).

"elasticsearch-local": {
  "command": "docker",
  "args": [
    "run", "-i", "--rm",
    "--add-host=host.docker.internal:host-gateway",
    "-e", "ES_URL", "-e", "ES_USERNAME", "-e", "ES_PASSWORD",
    "docker.elastic.co/mcp/elasticsearch:latest",
    "stdio"
  ],
  "env": {
    "ES_URL": "http://host.docker.internal:9200",
    "ES_USERNAME": "elastic",
    "ES_PASSWORD": "elastic_local"
  }
},

Qdrant

Векторный поиск и коллекции. API-ключ и веб-консоль на порту 6333.

Открыть Dashboard Документация
С хоста (db.local)
Базовый URL
http://db.local:6333
API-ключ (заголовок api-key)
qdrant_local
Контейнер в сети databases_net
Базовый URL
http://qdrant:6333
API-ключ
qdrant_local
Cursor MCP (~/.cursor/mcp.json)

MCP в Docker через uvx mcp-server-qdrant; первый запуск может быть долгим (зависимости и эмбеддинги). Том cursor-qdrant-mcp-cache в примере кэширует загрузки.

"qdrant-local": {
  "command": "docker",
  "args": [
    "run", "-i", "--rm",
    "--add-host=host.docker.internal:host-gateway",
    "-v", "cursor-qdrant-mcp-cache:/root/.cache",
    "-e", "QDRANT_URL", "-e", "QDRANT_API_KEY", "-e", "COLLECTION_NAME",
    "-e", "HF_HOME=/root/.cache/huggingface",
    "-e", "UV_CACHE_DIR=/root/.cache/uv",
    "ghcr.io/astral-sh/uv:python3.12-bookworm-slim",
    "uvx", "mcp-server-qdrant"
  ],
  "env": {
    "QDRANT_URL": "http://host.docker.internal:6333",
    "QDRANT_API_KEY": "qdrant_local",
    "COLLECTION_NAME": "mcp-local"
  }
},

Valkey

Совместим с Redis: ACL-пользователь и пароль; Redis Insight за HTTP Basic.

Открыть Redis Insight Документация
С хоста (db.local)
redis://valkey:valkey_local@db.local:6379
Контейнер в сети databases_net
redis://valkey:valkey_local@valkey:6379
Cursor MCP (~/.cursor/mcp.json)

MCP в Docker (Redis MCP Server); REDIS_HOST — host.docker.internal, пользователь ACL valkey (не default).

"valkey-local": {
  "command": "docker",
  "args": [
    "run", "-i", "--rm",
    "--add-host=host.docker.internal:host-gateway",
    "-e", "REDIS_HOST", "-e", "REDIS_PORT", "-e", "REDIS_USERNAME", "-e", "REDIS_PWD",
    "mcp/redis:latest"
  ],
  "env": {
    "REDIS_HOST": "host.docker.internal",
    "REDIS_PORT": "6379",
    "REDIS_USERNAME": "valkey",
    "REDIS_PWD": "valkey_local"
  }
},

Сеть databases_net

Чтобы из своего контейнера достучаться до СУБД по именам выше, подключите сервис к внешней сети этого стека. Сервер MCP_DOCKER (Docker MCP Toolkit) описан в README репозитория — он не привязан к одной СУБД.

Фрагмент docker-compose.yml
services:
  app:
    networks:
      - databases_net
    environment:
      DATABASE_URL: postgresql://postgres:postgres@postgres:5432/postgres
      MONGODB_URI: mongodb://admin:admin@mongodb:27017/?authSource=admin
      ELASTICSEARCH_URL: http://elastic:elastic_local@elasticsearch:9200
      REDIS_URL: redis://valkey:valkey_local@valkey:6379
      QDRANT_URL: http://qdrant:6333
      QDRANT_API_KEY: qdrant_local

networks:
  databases_net:
    external: true
Креды, домен db.local, hosts и порты — в README репозитория databases.
